So what really matters here?

Layout (width, height, top, left, font-size, ...)

Harms performance when recalculated frequently. Avoid forced synchronous layouts (offsetWidth, getComputedStyle, etc.)

Paint (background, box-shadow, ...)

Draws fills, borders, text, and shadows. Apparently harmless combinations can invalidate large areas. Shadows, blur, clipping, and complex backgrounds are often the first suspects.

Composition (transforms, opacity, ...)

Reassembles already rasterized layers, can create frames without blocking the main thread. But layers are not free: too many layers consume memory and raster time
