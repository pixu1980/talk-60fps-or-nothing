Wrapping up

CSS first

  • Great for state changes, entrances, scroll reveal, and choreography
  • If you stay on transform and opacity, the browser flies
  • New APIs now cover cases that used to require JS

JS when needed

  • Needed for realtime input, gestures, physics, and data orchestration
  • It should not fight the renderer with layout read and write work
  • Its best role is to provide intent not to move pixels "by hand"

The real comparison is not CSS vs JS. It's understanding which browser layer you are activating and whether your choice keeps the frame under 16.67ms.
